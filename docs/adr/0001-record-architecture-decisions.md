# 1. Record architecture decisions

Date: 2024-05-24

## Status
Accepted

## Context
We need a way to track the rationale behind technical choices, especially regarding the volatile DOM of AI Studio and the automation strategies used.

## Decision
We will use Architecture Decision Records (ADRs) as described by Michael Nygard. They will be stored in `docs/adr/`.

## Consequences
A historical record of technical choices is available for future maintainers.
